## 1 Linux 源码放在共享文件夹下编译失败

这种情况请使用 `wget` 等工具将 Linux 源码下载至容器内目录**而非共享目录**，然后执行编译。

## 2 QEMU+GDB 使用 `si` 单指令调试遇到模式切换失效

在遇到诸如 `mret`, `sret` 等指令造成的模式切换时，`si` 指令会失效，可能表现为程序开始不停跑，影响对程序运行行为的判断。

一个解决方法是在程序预期跳转的位置打上断点，比如：

```bash
(gdb) i r sepc    
0x8000babe
(gdb) b * 0x8000babe
(gdb) si    # 或者使用 c
```
